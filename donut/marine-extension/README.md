# Marine Chrome 扩展开发

日常开发不需要启动 Marine/Tauri，也不会下载 Wayfern。浏览器目标识别、字幕/评论/正文抓取、直评/回复高亮和上下文租约都在本目录内完成。

## 在 Chrome 中直接开发

1. 打开 `chrome://extensions`，开启“开发者模式”。
2. 选择“加载已解压的扩展程序”，目录指向本 `marine-extension` 文件夹。
3. 修改源码后，在 Marine 扩展卡片上点“重新加载”，再刷新正在调试的网页。
4. 聚焦直评框或某条评论的回复框。页面会显示 `Marine · 直评` 或 `Marine · 回复 @用户`，但扩展不会自动输入或发布。

页面识别和高亮可完全独立调试。需要把上下文交给 Rime 时，也可以启动仓库内的轻量开发桥，不必启动 Marine/Tauri：

```bash
pnpm dev:marine-extension-bridge
```

它会打印一组 `apiBase/token`，并为 Rime 写入仅本次进程有效的 `MarineDev/etinput-runtime.json`。把这两项填进扩展侧栏的“手动配置”，重新加载扩展页面即可。开发桥只返回固定调试草稿；可用 `MARINE_DEV_DRAFT="你的测试文字"` 自定义。它不会向网页输入或发布，按 `Ctrl-C` 会清理自己拥有的 runtime 文件。真实智能体生成仍在发布前用 Marine 桌面端联调。

连接真实 Marine 本地 API 时，独立 Chrome 还需要在同一配置区选择一个 Marine 身份。这个选择只决定发布记录归入哪个本地身份，不会切换网站账号。

## 发布记录

- Bilibili：扩展优先在 `/x/v2/reply/add` 返回 HTTP 成功、业务码为 `0` 且带有新评论 ID 时写入“平台确认已发布”；浏览器恢复既有标签页时会自动补注入捕获桥。若发布发生在扩展启动前，或页面使用了未被即时观察到的网络路径，扩展会从 Bilibili 自己的评论列表响应中对账本账号最近 7 天的评论。两条路径的最终文字、站点账号、发布时间、评论 ID 和层级都来自平台响应。
- 其他平台：话术卡片保留“手动确认已发”，记录前会要求确认实际发布文字；账本明确显示为“手动确认”，不会伪装成平台成功回执。
- AI 生成、填入网页编辑器和 Rime 上屏都不会自动算作发布；Marine 客户端的“评论账本”负责本地查询、筛选和统计。

## 测试

在仓库根目录运行：

```bash
pnpm test:marine-extension
MARINE_CHROME_HEADLESS=1 pnpm test:marine-extension:e2e
```

第一条覆盖上下文大小、竞态、标签页/窗口切换、Service Worker 状态持久化和轻量开发桥合同。第二条直接启动临时 Chrome，验证真实鼠标/键盘焦点事件、直评/回复高亮、稳定评论 ID、上下文撤销、SPA 导航清理，以及全程不输入、不提交。

## 与 Marine/Rime 的边界

- Chrome 扩展负责：识别当前页面与评论层级、抓取上下文、显示投放目标、维护焦点租约。
- Rime Action Plugin 负责：显式请求本地智能体、接收结果、校验结果仍绑定原目标，再放入缓冲区或收件箱。
- Marine 桌面端负责：生产环境的本地 API 与运行时能力令牌。Marine 真正退出后，Marine 动作应不可用。

“先开缓冲模式”与“先聚焦评论框”都允许；任何一边准备好时都不会自动触发生成，更不会自动发布。
